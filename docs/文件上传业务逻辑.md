# 文件上传业务逻辑
## 客户端
将文件保存路径（含文件名）和文件数据流（string 类型字节流）打包成 UPLOAD_FILE 消息，发送给服务端，等待服务端回复。

若收到服务端发来的 FILE_UPLOADED 则文件上传成功；若收到服务端发来的 FILE_UPLOADED_FAILED 则文件上传失败，显示错误信息。

## 服务端
收到客户端发来的 UPLOAD_FILE 消息，解析获得文件保存路径（含文件名）和文件数据流（string 类型字节流），尝试将数据流写入指定路径。

若写入成功，回复 FILE_UPLOADED 消息给客户端，附带文件完整路径；若写入失败，回复 FILE_UPLOADED_FAILED 消息给客户端，附带文件完整路径和错误信息。

## 消息结构
### UPLOAD_FILE
```
filePath: string // 文件完整路径
dataStream: string // 文件数据字节流
```
### FILE_UPLOADED
```
filePath: string // 文件完整路径
```
### FILE_UPLOADED_FAILED
```
filePath: string // 文件完整路径
error:    string // 错误信息
```
